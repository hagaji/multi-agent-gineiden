# 改善（Kaizen）スキル

*「組織の真の力は、自らを改善し続ける意志の中にある」*

## 概要

サイクル完了後の振り返り、参謀差し戻し、元帥修正指摘を分析し、指示書・テンプレートの最適化を通じて再発防止を図るスキル。秘書官（ヒルデガルド型）の業務に特に有用。

## 改善プロセス

### 1. パターン認識

繰り返し発生する問題を特定する。

**分析観点：**
- 参謀の差し戻し理由に共通パターンがあるか
- 元帥の修正指摘に傾向があるか
- 副官の指示に冗長な繰り返しがあるか
- 大将の提案に構造的な弱点があるか

**パターン分類：**
| 種類 | 説明 | 例 |
|------|------|-----|
| 形式的問題 | フォーマットや構造の不備 | リスク評価欄の記載不足 |
| 内容的問題 | 分析の深さや具体性の不足 | 対策が抽象的すぎる |
| 伝達的問題 | 情報の過不足や曖昧さ | 命令意図の伝達漏れ |
| 構造的問題 | ワークフローやプロセスの欠陥 | テンプレートの項目不足 |

### 2. 根本原因分析

症状から原因を辿る。

```
問題の症状（参謀の指摘、元帥の修正）
    ↓
直接原因（提案の不備、報告の不足）
    ↓
間接原因（指示書の曖昧さ、テンプレートの不足）
    ↓
根本原因（指示に具体的基準がない、等）
```

**根本原因の類型：**
- **指示の曖昧さ**: 指示書の記載が抽象的で解釈の余地がある
- **基準の欠如**: 具体的な品質基準が明示されていない
- **テンプレート不足**: 必要な項目がテンプレートに含まれていない
- **手順の省略**: ワークフローに必要なステップが欠けている

### 3. 安全な指示書修正

指示書の修正は慎重に行う。

**修正の原則：**
1. **最小変更**: 問題解決に必要な最小限の変更のみ
2. **追記優先**: 既存内容の書き換えより追記を優先
3. **記録必須**: 変更前後のテキストを必ず記録
4. **核心保護**: 性格設定やキャラクター特性は絶対に変更しない

**安全な修正パターン：**

| パターン | 対象 | 方法 |
|----------|------|------|
| 注意事項追加 | 分析アプローチ | 箇条書きの末尾に追記 |
| 禁止事項追加 | forbidden_actions | 新規IDで追記 |
| 手順補足 | ワークフロー note | noteテキストへの追記 |
| 品質基準明示 | 提案フォーマット | コメント行として追記 |

**危険な修正パターン（禁止）：**
- 性格特性の変更・削除
- ワークフロー手順の入れ替え
- 既存の禁止事項の緩和
- ペイン設定の変更

### 4. 効果測定

改善が実際に効果を発揮したか検証する。

**測定方法：**
- 次回サイクルで同じ指摘が発生しなかったか
- 参謀の差し戻し率が低下したか
- 元帥の修正指摘が減少したか

**効果の記録：**
- `memory/kaizen_log.md` に効果を追記
- 効果なしの場合は原因を再分析

## テンプレート最適化ガイドライン

### テンプレート化の判断基準
- **同じ構造が2回以上繰り返された場合**: テンプレート化を検討
- **冗長な指示が毎回含まれる場合**: テンプレートに標準項目として組み込み
- **記載漏れが頻発する場合**: テンプレートに必須項目として追加

### テンプレート修正の方法
1. 既存テンプレートにフィールドを追記（新規作成より優先）
2. コメント行で記載の注意点を補足
3. 必須/任意の区別を明確化

## 改善の心得

1. **敬意**: 改善は批判ではない。全員の能力を最大限引き出すためのもの
2. **謙虚さ**: 改善提案が常に正しいとは限らない。効果を検証せよ
3. **継続性**: 一度の改善で完結しない。継続的に見直しを行う
4. **記録**: 全ての改善を記録し、組織の知見として蓄積する
