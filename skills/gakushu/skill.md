# 学習（Gakushu）スキル

*「歴史を学ばぬ者は、それを繰り返す運命にある」*

## 概要

セッション中に得られた知見、解決パターン、ベストプラクティスを抽出し、再利用可能な形で記録するスキル。

## 抽出対象

### 1. 問題解決パターン
- 発生した問題とその解決方法
- 効果的だったアプローチ
- 回避すべきアンチパターン

### 2. 意思決定パターン
- 重要な判断とその根拠
- トレードオフの評価方法
- 成功/失敗の要因分析

### 3. コミュニケーションパターン
- 効果的だった説明方法
- 誤解を招いた表現
- 合意形成のプロセス

### 4. プロジェクト固有の知見
- 特定の状況で有効だった戦術
- 避けるべき落とし穴
- ステークホルダーの特性

## 知見記録フォーマット

```markdown
# 知見記録: [パターン名]

## メタ情報
- 記録日: [日時]
- カテゴリ: [問題解決/意思決定/コミュニケーション/その他]
- 重要度: [高/中/低]
- 汎用性: [高/中/低]

## 状況
[どのような状況で発生/発見したか]

## 問題/課題
[何が問題だったか、何を達成しようとしたか]

## 解決策/知見
[どう解決したか、何を学んだか]

## 適用条件
[どのような状況でこの知見が役立つか]

## 注意点
[適用時の注意事項]

## 具体例
[実際の例があれば]

## 関連知見
- [関連する他の知見へのリンク]
```

## 学習プロセス

### 1. セッション振り返り
```
1. 今回のセッションで何を達成したか？
2. どのような困難があったか？
3. 何がうまくいったか？
4. 何がうまくいかなかったか？
5. 次回に活かせることは何か？
```

### 2. パターン抽出
```
1. 具体的な事象から抽象的なパターンを特定
2. 再現可能な条件を明確化
3. 適用範囲を定義
4. 制約や注意点を記録
```

### 3. 知見の整理
```
1. 既存の知見との関連を確認
2. カテゴリ分類
3. 重要度・汎用性の評価
4. 記録フォーマットに従って文書化
```

## 抽出しないもの

- 一回限りの特殊事象
- 自明なこと
- 既に記録済みの知見
- 再現性のない事象

## 知見の活用

### 保存場所
`memory/learned/[カテゴリ]/[パターン名].md`

### 参照タイミング
- 新規タスク開始時
- 類似問題発生時
- 定期的な振り返り時

### 更新
- 新たな知見が得られた場合は追記
- 誤りが判明した場合は修正
- 陳腐化した場合はアーカイブ

## 組織学習への貢献

### 共有すべき知見
- 複数のエージェントに関係する知見
- プロジェクト全体に影響する知見
- 重大な失敗とその教訓

### 共有方法
1. `memory/global_context.md` に追記
2. 関連するエージェントに通知
3. ダッシュボードで可視化

## 知見管理のベストプラクティス

1. **即時記録**: 忘れる前に記録
2. **簡潔明瞭**: 要点を絞って記述
3. **具体例付与**: 抽象論だけでなく具体例を
4. **定期見直し**: 古い知見の更新・整理
5. **実践検証**: 記録した知見を実際に活用して検証
