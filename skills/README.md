# スキル一覧

銀河英雄伝説マルチエージェントシステムで使用可能なスキル。

## 利用可能なスキル

### /keikaku（計画）
**用途**: 複雑なタスクの戦略立案

```
元帥の命令を分析し、実行計画を策定する。
- 要件分析
- アプローチ比較（保守的 vs 挑戦的）
- リスク評価
- 実行フェーズの設計
```

**使用タイミング**:
- 新規タスク開始時
- 複数のアプローチが考えられる場合
- リスク評価が必要な場合

### /kensho（検証）
**用途**: 提案・成果物の品質検証

```
提案の妥当性を多角的に検証する。
- 命令整合性チェック
- 論理的整合性検証
- 実現可能性評価
- リスク分析
```

**使用タイミング**:
- 参謀による提案レビュー時
- 最終報告前の品質確認
- 重要な意思決定前

### /gakushu（学習）
**用途**: 知見の抽出と蓄積

```
セッションから再利用可能なパターンを抽出。
- 問題解決パターン
- 意思決定パターン
- ベストプラクティス
```

**使用タイミング**:
- 問題解決後の振り返り
- セッション終了時
- 重要な発見があった時

### /checkpoint（チェックポイント）
**用途**: 進捗記録と状態保存

```
作業の節目で状態を記録し、復帰点を確保。
- 進捗の可視化
- 復帰点の確保
- 引き継ぎ情報の整理
```

**使用タイミング**:
- フェーズ完了時
- リスクの高い操作前
- セッション終了前
- コンテキスト圧縮前

### /kaizen（改善）
**用途**: システム改善・プロンプト最適化

```
サイクルの振り返りからシステム改善点を特定し、
指示書・テンプレートを最適化する。
- パターン認識
- 根本原因分析
- 安全な指示書修正
- 効果測定
```

**使用タイミング**:
- サイクル完了後の振り返り
- 参謀による差し戻し発生時
- 元帥からの修正指摘時


## スキルの使い方

各エージェントは、自分の指示書に記載されたタイミングでスキルを参照・活用します。

### 手動呼び出し
```
スキル「keikaku」を参照して、この命令の計画を立案せよ。
```

### 自動適用
指示書にスキル参照を含めることで、特定の状況で自動的に適用されます。

## スキル追加方法

1. `skills/[スキル名]/` ディレクトリを作成
2. `skill.md` ファイルを作成
3. YAML フロントマターでメタ情報を定義
4. スキル内容を記述
5. この README を更新
